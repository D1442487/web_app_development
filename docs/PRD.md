# 產品需求文件 (PRD)：大學生校園活動報名系統

## 1. 產品簡介 (Product Overview)
本系統為專為「大學生」設計的活動報名平台。旨在提供一個「簡潔有力、直觀不複雜」的操作體驗，讓活動主辦方（如系學會、社團）能夠快速建立活動，並讓學生輕鬆完成報名手續，解決傳統紙本或 Google 表單不易即時控管人數、需要繁瑣人工對帳的痛點。

## 2. 目標受眾 (Target Audience)
*   **前台使用者（大學生）**：想報名參加校園活動、講座或社團聚會的學生，期望有快速、清晰、支援手機操作的報名流程。
*   **後台管理者（活動主辦方）**：社團幹部、系學會成員等，需要建立活動表單、管理報名資料與追蹤報名狀況。

## 3. 核心功能需求 (MVP Features)

這 5 項重點功能構築了系統的最小可行性產品（MVP），確保滿足主要的報名與管理需求：

### 3.1 [建立活動] (Create Event)
*   **功能描述**：主辦方可透過後台快速建立新的活動頁面。
*   **必填欄位**：活動名稱、活動日期與時間、活動地點、簡介/注意事項、**報名人數上限**。
*   **預期行為**：建立成功後，系統會自動生成專屬的活動報名網址，主辦方可直接複製連結發布於 IG、FB 或 LINE。

### 3.2 [活動報名人數] (Registration Headcount)
*   **功能描述**：即時統計並顯示目前的報名人數。
*   **前台顯示**：前台報名頁面能動態顯示「目前已報名人數 / 總人數上限」（如：報名進度 45/50 人），提供參與者透明資訊，並可運用稀缺性加速報名進度。
*   **後台顯示**：管理者在後台活動列表能一眼掌握各活動的招募進度與狀態。

### 3.3 [活動報名資料] (Registration Data)
*   **功能描述**：系統化收集與儲存報名者的個人資訊。
*   **報名欄位**：預設包含「姓名、學號、系級、聯絡電話、Email」。
*   **預期行為**：報名資料需結構化存放，後台需提供以列表方式檢視所有報名者資訊的功能，並具備「匯出成 CSV / Excel 檔案」的按鈕，方便主辦方後續點名手續或寄送行前通知。

### 3.4 [管理者後台] (Admin Dashboard)
*   **功能描述**：一個給主辦方專屬且介面清晰簡易的中央控制面板。
*   **功能包含**：
    *   **活動總覽**：條列所有「進行中」、「已結束」的活動狀態。
    *   **活動管理**：針對單一活動進行編輯、手動強制關閉/開放報名等操作。
    *   **成效檢視**：每個活動都有獨立卡片或操作區塊，點擊即可切換至該活動的「報名資料」名單頁面。

### 3.5 [額滿自動關閉按鈕] (Auto-close on Full Capacity)
*   **功能描述**：自動化滿額控管防呆機制，避免超額報名引發後續確認與退款的糾紛。
*   **觸發條件**：當系統判斷「已送出的成功報名數」達到了「建立活動時設定的人數上限」。
*   **系統行為**：
    *   前台報名頁面的「提交報名」按鈕自動反灰並改為顯示「已額滿 (Full)」，禁止點擊。
    *   系統後端拒絕接收新的報名請求，並在畫面上給出友善的滿員提示。

## 4. 使用者流程規劃 (User Flow)

### 大學生 (前台報名流程)
1. 點擊主辦方給的活動專屬網址進入頁面。
2. 查看活動詳情與目前剩餘名額狀況。
3. 若尚未額滿，填寫學號、姓名等必要報名資料。
4. 點擊「送出報名」，畫面跳轉至大大的「報名成功」提示頁面（可引導截圖保存）。

### 主辦方 (後台管理流程)
1. 登入管理者後台。
2. 點擊「建立活動」，輸入活動相關資訊與設定人數上限，並送出。
3. 取得報名連結開始對外宣傳。
4. 活動進行期間隨時登入後台查看「報名人數」與「報名資料」。
5. 報名額滿時，系統自動關閉前台報名；主辦方亦可隨時手動提早關閉。
6. 將最終名單匯出為 Excel 供當天驗票與報到使用。

## 5. UI/UX 與技術非功能需求
*   **行動裝置優先 (Mobile-First)**：介面設計需針對手機版高度優化，因為大學生幾乎只會用手機點開 IG 連結填寫報名表。
*   **免安裝與免前台註冊帳號**：為了做到「直觀不複雜」，報名者無需下載 App 或註冊平台會員，藉由填寫「學號」作為系統識別即可直接報名，大幅降低報名門檻。
*   **極簡視覺風格**：採現代化設計（如圓角大卡片、顯眼的主按鈕、大量的留白），符合年輕人審美，並幫助操作直覺化。
